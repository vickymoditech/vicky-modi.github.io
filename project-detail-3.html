Capauth: Django SSO Backend

Organisation Management:

  • Organisation Visibility: All users can view all organisations, with potential for future modifications to restrict visibility.
  • Organisation Creation: Users can create new organisations, automatically becoming admins of the organisations they create.
  • Default Organisation: New users are initially assigned to a default organisation before creating or joining others.
  • Edit and Delete Control: Only the creator of an organisation can edit or delete it.

User Management:

  • Intra-Organisation Access: Users within an organisation can access information about other users in the same organisation.
  • User Addition Process:
    1. Check if the new user belongs to the same company.
    2. Verify if the user already exists in the system.
    3. If present, assign the new organisation to the existing user.
    4. If not present, create a new user, assign to the default organisation, then add to the specified organisation.
  • Multi-System Registration: Users need to register in Cap-ai and other integrated systems.
  • Edit and Delete Permissions: Only admins or the user themselves can edit or delete user information.

Technologies Used:

  • Back-end: Django for robust server-side functionality.
  • Database: PostgreSQL for flexible data storage and retrieval.
  • Authentication: Django Oauth Toolkit for secure user login and access control.
